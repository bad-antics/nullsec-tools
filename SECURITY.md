## Reporting a Vulnerability

If you discover a security vulnerability within NullSec Tools, please follow these steps:

### 1. Do NOT Open a Public Issue

Security vulnerabilities should be reported privately to prevent exploitation.

### 2. Contact Us

Send an email to the maintainers or use GitHub's private vulnerability reporting:

- **GitHub:** Open a private security advisory
- **PGP:** Available upon request for encrypted communication

### 3. Include Details

Please provide:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### 4. Response Timeline

- **Initial Response:** Within 48 hours
- **Status Update:** Within 7 days
- **Resolution:** Dependent on severity

## Security Best Practices

When using NullSec Tools:

1. **Keep Updated:** Always use the latest version
2. **Verify Downloads:** Check file hashes
3. **Isolated Environment:** Use VMs or containers for testing
4. **Authorization:** Only test authorized systems
5. **Logging:** Maintain records of all testing activities

---

## Responsible Disclosure

We follow responsible disclosure practices:

1. Vulnerabilities are fixed before public disclosure
2. Credit is given to reporters (unless anonymity is requested)
3. CVEs are assigned for significant vulnerabilities

---

## Known Security Considerations

These tools are **designed for security testing** and include:

- Network scanning capabilities
- Password cracking wrappers
- Payload generation
- System enumeration

**These are features, not vulnerabilities.**
